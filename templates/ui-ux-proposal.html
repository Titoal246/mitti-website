Design Handoff Checklist

A thorough handoff prevents "why does this look different in development?" disputes. Share this checklist with your client at handoff:

  • Figma file shared with view access to developer team
  • All layers named and organised (no "Frame 1", "Rectangle 2")
  • Components created for all repeating elements
  • States defined: default, hover, active, disabled, loading, error, empty
  • Colours using named styles (not hardcoded hex)
  • Typography using text styles (not manual font settings)
  • Spacing uses 8px grid system (documented)
  • Icons exported as SVG with correct viewBox
  • Image placeholders with correct aspect ratios
  • Annotations on complex interactions or edge cases
  • Prototype link shared for flow walkthroughs
  • Assets exported to Drive/WeTransfer folder

How to Structure Discovery Pricing

Many designers skip discovery and go straight to screens — this is the primary cause of redesigns and scope creep. Discovery is where you earn the right to design confidently. Price it properly:

  • Under ₹30,000 total project: Discovery is bundled — include 1 kickoff call + competitive analysis in the project price
  • ₹30,000–₹1,50,000 project: Separate discovery as 15–20% of total project fee (e.g., ₹15,000 discovery on a ₹1,00,000 project)
  • Above ₹1,50,000: Sell discovery as a standalone engagement first. Many clients who "just want screens" discover real user problems during discovery and expand scope willingly.
Discovery as a sales tool: A paid discovery phase signals serious clients (low-commitment clients walk away). It also positions you as a strategic partner, not just a screen-pusher — which justifies higher visual design rates.
